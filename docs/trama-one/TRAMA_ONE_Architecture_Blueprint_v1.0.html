Una sola trama. Tre portali. Un dominio condiviso.

TRAMA ONE è la nuova esperienza parallela che unifica Parent, Partner e Admin senza riscrivere l’applicazione. Mantiene Legacy e Next Gen, introduce una dorsale end-to-end misurabile e usa dati canonici condivisi.

State machine condivise

Le UI non definiscono gli stati: li rappresentano. Le transizioni appartengono al dominio.

Partner

DRAFT→SUBMITTED→IN_REVIEW→APPROVED→ONBOARDING→ACTIVE

Alternative: NEEDS_INFO · REJECTED · SUSPENDED

Activity

DRAFT→READY→IN_REVIEW→PUBLISHED

Alternative: CHANGES_REQUIRED · PAUSED · ARCHIVED

Request

DRAFT→SUBMITTED→PENDING_PARTNER→ACCEPTED→CONFIRMED

Alternative: COUNTER_PROPOSED · DECLINED · EXPIRED · CANCELLED

Referral

REPORTED→QUALIFIED→INVITED→CLAIMED→ACTIVATED

Reward: ELIGIBLE → EARNED → REDEEMED / REVERSED

Quality gate

TRAMA ONE è pronta per la beta solo quando prodotto, dati e operazioni sono verificabili insieme.

Product

  • CTA dominante e prossima azione chiara
  • Contesto preservato
  • Empty/error/retry state
  • Feedback contestuale

Technology

  • Build, lint e typecheck
  • RLS/RBAC verificati
  • State machine canoniche
  • Feature flag e rollback

Operations

  • SLA e audit
  • Partner verificati
  • Disponibilità aggiornata
  • Test E2E cross-portale
